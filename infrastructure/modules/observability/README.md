# Observability Module

This module provisions CloudWatch dashboards and alarms for the semantic search stack. It consumes identifiers published by the runtime, ingestion, and vector-store modules to render time-series metrics, logs, and SLO guardrails without hard-coding resource names.

## Features

- Optional CloudWatch dashboard featuring:
  - Search runtime P95 latency (custom metric `SemanticSearch/Runtime::QueryLatencyP95`)
  - Search runtime error rate (`SemanticSearch/Runtime::QueryErrorRate`)
  - Ingestion queue depth (SQS `ApproximateNumberOfMessagesVisible`)
  - Log Insights widgets for any supplied log groups
- Optional CloudWatch alarms with overridable thresholds for:
  - Search runtime P95 latency breaches
  - Search runtime error rate spikes
- Support for notifying an SNS topic when alarms enter ALARM or OK state
- Configurable log queries and dashboard timeframes

## Usage

```hcl
module "observability" {
  source = "../../modules/observability"

  project     = var.project
  environment = var.environment
  tags        = local.common_tags

  metrics_sources = {
    search_service  = module.search_service_fargate[0].service_name
    ingestion_queue = module.data_plane.ingestion_queue_arn
  }

  log_group_names = {
    runtime = module.search_service_fargate[0].log_group_name
    api     = module.search_service_lambda[0].api_log_group_name
  }

  widget_queries = {
    runtime = "fields @timestamp, level, request_id, message | sort @timestamp desc | limit 50"
  }

  notification_topic_arn = aws_sns_topic.alerts.arn
}
```

> **Tip:** When using the Lambda runtime, set `metrics_sources.search_service` to the Lambda function name instead of the ECS service name.

## Inputs

| Name | Type | Description |
|------|------|-------------|
| `project` | `string` | Project identifier used for tagging and naming resources. |
| `environment` | `string` | Deployment environment label (e.g., `dev`, `staging`, `prod`). |
| `name_prefix` | `string` (optional) | Overrides the default `<project>-<environment>-observability` prefix. |
| `tags` | `map(string)` | Additional tags applied to all resources. |
| `metrics_sources` | `object` | Optional identifiers for telemetry sources: `search_service`, `ingestion_queue`, `vector_store`, `embedding_job`. Accepts names or ARNs. |
| `enable_dashboards` | `bool` | Enables CloudWatch dashboard creation. Default `true`. |
| `enable_alarms` | `bool` | Enables CloudWatch alarms. Default `true`. |
| `dashboard_timeframe_hours` | `number` | Default dashboard time range (hours). Default `24`. |
| `alarm_thresholds` | `object` | Optional overrides for alarm thresholds (`search_latency_p95`, `search_error_rate`, `lambda_throttles`, `ecs_unhealthy_host_count`). |
| `notification_topic_arn` | `string` | SNS topic ARN for alarm notifications. Leave empty to suppress notifications. |
| `log_group_names` | `map(string)` | Optional mapping of logical names to CloudWatch log group names for Log Insights widgets. |
| `widget_queries` | `map(string)` | Optional custom Logs Insights queries keyed by the same identifiers used in `log_group_names`. |

## Outputs

| Name | Description |
|------|-------------|
| `dashboard_name` | CloudWatch dashboard name (or `null` if dashboards are disabled or no widgets were created). |
| `alarm_names` | List of alarm names generated by the module. |
| `alarm_arns` | List of alarm ARNs generated by the module. |

## Implementation Notes

- Custom runtime metrics (`QueryLatencyP95`, `QueryErrorRate`) must be emitted by the application for the dashboard and alarms to function as expected.
- When `metrics_sources.ingestion_queue` is an ARN, the module extracts the queue name automatically for SQS metrics.
- Log widgets default to a simple descending log view if no custom query is supplied.
- Alarms treat missing data as non-breaching to avoid flapping during quiet periods; adjust this behaviour by forking the module if stricter semantics are required.
